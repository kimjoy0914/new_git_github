# 강사의 특정 repository commit 이력 없이 가져가기
왜 Commit 이력을 제거할까?
훈련 현장에서 다음 상황이 자주 발생합니다.

예시 상황
강사 Repository

spring-react-erp
 ├─ backend
 ├─ frontend
 ├─ docker-compose.yml
 ├─ README.md

훈련생이 그대로 clone 하면 다음 문제가 생깁니다.

문제
git clone https://github.com/teacher/erp-project

그러면
commit 130개
작성자: 강사

즉
✔ 모든 commit 기록이 그대로 따라옴
✔ 학생 프로젝트가 아니라 강사 프로젝트 기록이 됨
따라서 교육에서는 보통
코드만 가져오고 Git 이력은 새로 시작합니다.

방법 1 (가장 많이 사용하는 방법)
GitHub ZIP 다운로드
1️⃣ Repository 접속
예
https://github.com/teacher/erp-project
2️⃣ Code 버튼 클릭
Code
 ├ HTTPS
 ├ SSH
 └ Download ZIP
3️⃣ Download ZIP

압축을 풀면
erp-project-main
 ├ backend
 ├ frontend
 ├ README.md
이 상태에서는
✔ git 정보 없음
✔ commit 이력 없음
======================================================================
방법 2 (실무에서 많이 사용하는 방법)
.git 폴더 삭제
강사가 clone을 하라고 한 경우
git clone https://github.com/teacher/erp-project

폴더 구조
erp-project
 ├ .git
 ├ backend
 ├ frontend

여기서
.git
폴더가 commit 기록 저장소입니다.

1️⃣ .git 삭제
Linux / Mac
rm -rf .git
Windows
rmdir /s .git
2️⃣ 새로운 git 초기화
git init
3️⃣ commit 생성
git add .
git commit -m "first commit"

이제
✔ 기존 commit 기록 없음
✔ 완전히 새로운 repository

# 강사의 특정 repository commit 다 유지한채 가져가기

2. 실습 배경
실무에서는 다음과 같은 상황이 많습니다.

예시
회사 프로젝트
erp-system
 ├ backend
 ├ frontend
 ├ docker-compose.yml
 └ README.md

이 프로젝트는 이미
commit 250개
작성자: 여러 개발자
가 존재합니다.

새로운 개발자가 합류하면
👉 기존 commit 기록을 모두 유지한 채 프로젝트를 가져옵니다.

3. Git Clone이란?
Git clone은 단순히 파일을 복사하는 것이 아닙니다.
다음을 모두 가져옵니다.

프로젝트 코드
+
commit 이력
+
branch 정보
+
tag 정보

즉 전체 개발 히스토리 복사 입니다.

4. 기본 실습
Repository Clone
1️⃣ GitHub Repository 주소 확인

예
https://github.com/teacher/spring-react-erp

2️⃣ clone 실행
git clone https://github.com/teacher/spring-react-erp

실행 결과
spring-react-erp
 ├ backend
 ├ frontend
 ├ docker-compose.yml
 ├ README.md
 └ .git

여기서 중요한 폴더
.git

이 폴더 안에
commit history
branch 정보
tag 정보
가 저장되어 있습니다.

5. commit 이력 확인
프로젝트 폴더 이동
cd spring-react-erp
commit 로그 확인

git log

예시 결과
commit 9a31f2c
Author: teacher
Date: ...

Add JWT authentication

commit 8b2d4f1
Author: teacher
Date: ...

Create user API

commit 72adf33
Author: teacher
Date: ...

Initial project setup

✔ 개발 진행 과정
✔ 기능 추가 흐름
✔ commit 메시지 작성 방식

을 학습할 수 있습니다.
